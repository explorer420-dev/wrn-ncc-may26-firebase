Section 1
Before the Event — Importing Attendee Data
📋
The master attendee list lives in a Google Sheet. Every morning before the event opens, run the import to push the latest registrations into the system.

DDay 2FALSEOne column per event day
E–HDay 3–6…Same pattern
Running the Import

  1. Open the Google Sheet that has the registration data.

  2. In the menu bar click Extensions → Apps Script.

  3. Select the importToFirestore function from the dropdown at the top.

  4. Click Run (▶). Approve permissions if prompted (first time only).

  5. Wait for the execution log to show "Import complete". This typically takes 1–2 minutes for a few hundred rows.

  6. Verify in admin.html → Registered tab that the count matches your sheet.

⏰
When to run: Once each morning before the venue opens. If late registrations come in during the day, re-run the import — it is safe to run multiple times, existing attendance records are not overwritten.
Section 2
Event Day — Desk Check-in
Opening the App

  1. Open index.html in the browser on the check-in device.

  2. Sign in with the desk credentials (email + password shared by the admin).

  3. The app loads the current event day automatically. Confirm the day shown in the banner is correct.

  4. You're ready. The phone number field is focused and ready for input.

Checking In an Attendee

  1. Ask the attendee for their registered mobile number.

  2. Type the 10-digit number — the app looks up the record as you type.

  3. The screen shows the attendee's name and check-in status. Confirm with the attendee that the name matches.

  4. Press Confirm Check-in. The screen briefly shows a success confirmation, then resets for the next attendee.

✓
The app resets automatically after each check-in. No need to press back or refresh.
Day-of Checklist
  • Device is charged / plugged in
  • Stable internet connection confirmed
  • Import has been run this morning
  • Desk credentials available (email + password)
  • Correct day shown in the app banner
  • Walk-in payment process confirmed with team

Event Day — Admin Dashboard
Accessing the Dashboard

  1. Open admin.html on the admin device.

  2. Sign in with the admin credentials (separate from the desk credentials).

  3. The dashboard loads with live statistics and two data tabs.

Section 5
Exporting Data

  1. Open admin.html and sign in.

  2. Go to the tab you want to export — Registered or Attendance Log.

  3. Optionally use the search box to filter before exporting (only filtered rows are exported).

  4. Click ⬇ Download CSV. The file downloads immediately to your device.

  5. Open in Excel or Google Sheets for further analysis.

Section 6
Troubleshooting
ProblemLikely causeFix
Login page won't proceed Wrong credentials or no internet Check credentials with admin; check network connection
Attendee not found Phone number not in master, or import not run today Check the number carefully; re-run import if registrations came in after last import
Dashboard shows old data Import not run yet today Run the Apps Script import from Google Sheets
App shows wrong day Real date outside event window, or override set incorrectly Check test_day_override in Firestore config/event; set to 0 for real date
Check-in not appearing in log Network dropped mid-check-in Check internet; retry the check-in (duplicate protection is built in)
Page auto-logs in as wrong user Browser session from a different device/user Click Sign out (if visible) or open in an incognito/private window
